WebSocket 1005 Connection Drops: I discovered why the server was abruptly closing the connection. When the Gemini AI issues multiple tool calls simultaneously (e.g. log_sales_insight alongside research_competitor), the backend proxy was attempting to send multiple separate response payloads using a loop. The Gemini protocol strictly requires all function responses for a single turn to be batched in one single sendToolResponse array. Submitting them one by one was causing a protocol violation which instantly crashed the connection. I've updated apps/api/src/services/gemini-live.ts to accumulate the responses into an array and send them as one grouped payload.
